# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Repo layout

- `ui/quizzler/` — Angular 18 frontend.
- `api/quizzler/` — Spring Boot 2.7.18 backend (Java 17, Maven, PostgreSQL, JPA on `javax.persistence`). **Stays on Spring Boot 2.x** because the project mandates `javax.persistence` (no Jakarta).
- `product/` — non-code product artifacts: `requirements.md`, arc42 architecture template in `product/architecture/`, user stories in `product/project/` (e.g. `US_1.md`, `stories.md`, `TODO.md`). Read these for feature context before implementing user-story branches like `feature/US_1`.

## Frontend commands (run from `ui/quizzler/`)

- `npm start` — dev server at `http://localhost:4200/`
- `npm run build` — production build into `dist/quizzler`
- `npm test` — Jest (not Karma, despite the stale README). Uses `jest-preset-angular`.
- `npm run test:watch` / `npm run test:coverage`
- Run a single test: `npx jest path/to/file.spec.ts` or `npx jest -t "test name pattern"`
- `npm run lint` — angular-eslint via the Angular CLI builder (`ng lint`)

The top-level `README.md` was generated by Angular CLI and is partially stale (mentions Karma/`ng test`; the project uses Jest).

## Backend commands (run from `api/quizzler/`)

- `mvn spring-boot:run` — start the API on `http://localhost:8080`. Requires a Postgres instance reachable at `localhost:5432` with DB `quizzler` (default user/pass `quizzler`/`quizzler`).
- `mvn -DskipTests package` — build the jar.
- `mvn test` — run tests.

Seed data is inserted on first startup by `SeedDataInitializer` (idempotent — skipped if any question exists).

## Frontend architecture

Standalone-component Angular app, no NgModules. Bootstrapping path: `src/main.ts` → `app.config.ts` (provides `provideRouter(routes)` and `provideHttpClient()`) → `app.component.ts` (hosts `<router-outlet>`) → routes in `app.routes.ts`.

Domain layering inside `src/app/`:
- `entities/` — plain TypeScript data classes (`SinglePickQuestion`, `SingePickOption`, `SinglePickResult`). Used as both HTTP response types and form models.
- `services/questionservice.ts` — `@Injectable({ providedIn: 'root' })`. `getSinglePickQuestionById` hits `GET ${apiBaseUrl}/question/{id}` and returns `Observable<SinglePickQuestion>`. `evaluate` is still a client-side stub pending a `/evaluate` endpoint.
- `singlepick/` — feature component using Reactive Forms. Loads the question asynchronously and stores it in an optional `singlePickQuestion?` field. The template is wrapped in `*ngIf="singlePickQuestion"` to handle the load delay. Post-evaluation highlighting via `correctOption` (`-1` = un-evaluated; otherwise the correct option id). The template uses `[ngClass]` to switch between `selected` / `correct` / `false` states.

## Backend architecture

Layered Spring Boot app under `com.quizzler.api`:

- `controller/QuestionController` — `GET /question/{questionId}` returns a `SinglePickQuestionDto`.
- `service/QuestionService` — fetches via repo, raises `ResponseStatusException(NOT_FOUND)` if missing or wrong type, maps to DTO.
- `repository/QuestionRepository` — `JpaRepository<Question, Long>` over the base type.
- `domain/` — `Question` (abstract, `@Inheritance(JOINED)`, `@DiscriminatorColumn("type")`), `SinglePickQuestion` (extends `Question`, `@DiscriminatorValue("SINGLE_PICK")`, owns `options` via `@OneToMany` + `@JoinColumn`, has `correctOptionId`), `SinglePickOption` (own table).
- `dto/` — wire-format classes; **`SinglePickQuestionDto` deliberately omits `correctOptionId`** so the answer never leaks to the client. Contains a `type` discriminator field for forward-compat with future question types.
- `config/WebConfig` — CORS, opens `http://localhost:4200`.
- `bootstrap/SeedDataInitializer` — `CommandLineRunner` that seeds question id 1 (the one referenced by the Angular flow) on a fresh database.

When adding new question types: subclass `Question` with a new `@DiscriminatorValue`, add a DTO, and extend the service mapping. Do **not** return entities directly from controllers — always map to DTOs.

## Conventions enforced by lint

`eslint.config.js` requires:
- Component selectors: element form, **kebab-case**, prefix **`quizzler-`** (e.g. `quizzler-singlepick`).
- Directive selectors: attribute form, **camelCase**, prefix **`quizzler`**.

The Angular CLI `prefix` in `angular.json` is also `quizzler` — keep these in sync when scaffolding.

## Git workflow

Branches are named per user story, e.g. `feature/US_1`, with commit messages prefixed `feature US_<n>: <subject>`. Main branch is `master`.

## Iteration scope

`US_1` (`product/project/US_1.md`) is the **first iteration** in its entirety: one participant, one single-pick question, select-and-submit, show correct answer. Treat anything beyond that flow (cross-question navigation, persistence, auth, the Spring Boot/PostgreSQL backend) as future iterations unless the user says otherwise.
